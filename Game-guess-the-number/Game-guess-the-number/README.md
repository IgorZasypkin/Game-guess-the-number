1. Принцип единственной ответственности (Single Responsibility Principle - SRP)
Что сделано:

Каждый класс имеет только одну причину для изменения:

RandomNumberGenerator - только генерация случайных чисел

ConsoleUserInput - только обработка пользовательского ввода

ConsoleOutput - только вывод информации

GameLogic - только логика игры (проверка догадок, подсчет попыток)

GuessNumberGame - только управление игровым процессом


2. Принцип открытости/закрытости (Open/Closed Principle - OCP)
Что сделано:

Классы открыты для расширения (через интерфейсы), но закрыты для модификации


3. Принцип подстановки Барбары Лисков (Liskov Substitution Principle - LSP)
Что сделано:

Производные классы могут заменять базовые, не ломая функциональность


4. Принцип разделения интерфейсов (Interface Segregation Principle - ISP)
Что сделано:

Интерфейсы небольшие и специфичные


5. Принцип инверсии зависимостей (Dependency Inversion Principle - DIP)
Что сделано:

Зависимости внедряются через конструктор (Dependency Injection)
